JSON 数据结构帮助文档

概述

stationData 是一个 JSON 对象，它代表了地铁线路及其相关站点的信息。 以下是 JSON 的主要结构和每个字段的详细说明。

JSON 结构


{
	"line_name_cn": "1号线",
	"line_name_en": "Line 1",
	"line_color": "#FF2023",
	"current_door": 2,
	"stations": [
		{
			"name_cn": "站A",
			"name_en": "Station A",
			"time_to_previous": 0,
			"transfers": [
				{
					"name": "日光",
					"color": "#1E90FF"
				}
			],
			"turns": [],
			"doors": [2, 3, 3, 4, 6, 6, 2]
		},
		// 更多站点...
	],
	"startpointId": 7,
	"endpointId": 0
}

字段说明

  • line_name_cn (字符串)：
    地铁线路的中文名称，例如 "1号线"。
  • line_name_en (字符串)：
    地铁线路的英文名称，例如 "Line 1"。
  • line_color (字符串)：
    代表地铁线路的颜色，使用十六进制表示法，例如 "#FF2023"。
  • current_door (整数)：
    当前打开的车门编号，用数字表示。
  • stations (数组)：
    一个包含所有站点信息的数组，每个站点是一个对象，包含以下字段：
    • name_cn (字符串)：本站的中文名称，例如 "站A"。
    • name_en (字符串)：本站的英文名称，例如 "Station A"。
    • time_to_previous (整数)：到达前一个站点所需的时间（单位：分钟）。
    • transfers (数组)：
      一个包含所有换乘线路信息的数组，每个换乘信息是一个对象，包含以下字段：
      • name (字符串)：换乘线路的名称，例如 "日光"。
      • color (字符串)：换乘线路的颜色，使用十六进制表示法，例如 "#1E90FF"。
    • turns (数组)：
      一个包含所有转乘信息的数组，每个转乘信息是一个对象，包含以下字段：
      • name (字符串)：转乘线路或枢纽的名称，例如 "576"。
      • exit (整数)：从本站转乘后的出口编号。
      • svg (24*24像素svg字符串)：用于描述转乘枢纽的图标(可选字段)。
    • doors (数组)：
      车站的每一个出口对应最近的车门的排列情况，数组中的数字代表门的编号，例如 [2, 3, 3, 4, 6, 6, 2]。
  • startpointId (整数)：
    起点站的 ID，通常用于标识线路的起始站。
  • endpointId (整数)：
    终点站的 ID，通常用于标识线路的终点站。

注意事项

在构建 JSON 时，请确保所有字符串都用双引号括起来，并注意语法的正确性（例如，键与值之间用冒号分隔，每个键值对用逗号分隔）。 数组元素之间也必须用逗号分隔，最后一个元素后面不得有多余的逗号。
